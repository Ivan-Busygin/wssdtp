# Contributing to WSSDTP

Thank you for your interest in contributing to WSSDTP! We appreciate your help in improving this project.

## Code of Conduct

Please be respectful and constructive in all interactions. We strive to provide a welcoming and inclusive environment for all participants.

## How to Contribute

### Reporting Bugs

Before creating a bug report, please check the issue list to ensure the bug hasn't been reported already. When creating a bug report, include:

- A clear title and description
- Precise steps to reproduce the problem
- Expected behavior and actual behavior
- Your environment (Go version, OS, etc.)
- Any relevant logs or error messages

### Feature Suggestions

Feature suggestions are welcome! Please:

- Use a clear and descriptive title
- Provide a detailed description of the proposed improvement
- Explain the current behavior and desired behavior
- Include use cases or examples

### Pull Requests

1. Fork the repository and create a feature branch from `main`
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. Make changes and ensure they follow the project's coding style

3. Write or update tests to cover your changes

4. Run tests locally:
   ```bash
   go test ./...
   ```

5. Commit changes with clear commit messages
   ```bash
   git commit -m "Add clear description of changes"
   ```

6. Push and create a pull request with a detailed description of the changes

### Code Style

- Follow standard Go conventions
- Use `gofmt` to format code
- Add comments for exported functions and types
- Write clear and meaningful variable and function names

### Testing

- Add tests for new features
- Ensure existing tests pass
- Test on different platforms if possible

### Documentation

- Update documentation when changing APIs
- Add usage examples for new features
- Keep README.md up to date

## Release Process

- Changes are reflected in version numbers as needed
- Tested for compatibility

## License

By contributing to WSSDTP, you agree that your contributions will be licensed under the same license as the project.

## Questions?

Feel free to open an issue if you have questions or need clarification on the contribution guidelines.

# Внесение вклада в WSSDTP

Спасибо за интерес к внесению вклада в WSSDTP! Мы ценим вашу помощь в улучшении этого проекта.

## Кодекс поведения

Пожалуйста, будьте уважительны и конструктивны во всех взаимодействиях. Мы стремимся предоставить приветливую и инклюзивную среду для всех участников.

## Как внести вклад

### Сообщение об ошибках

Перед созданием отчета об ошибке, пожалуйста, проверьте список issues, чтобы убедиться, что ошибка еще не была сообщена. Когда вы создаете отчет об ошибке, включите:

- Четкий заголовок и описание
- Точные шаги для воспроизведения проблемы
- Ожидаемое поведение и фактическое поведение
- Вашу среду (версия Go, ОС и т.д.)
- Любые релевантные логи или сообщения об ошибках

### Предложения улучшений

Предложения улучшений приветствуются! Пожалуйста:

- Используйте четкий и описательный заголовок
- Предоставьте подробное описание предлагаемого улучшения
- Объясните текущее поведение и желаемое поведение
- Включите случаи использования или примеры

### Pull Requests

1. Сделайте форк репозитория и создайте ветку feature от `main`
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. Внесите изменения и убедитесь, что они следуют стилю кода проекта

3. Напишите или обновите тесты для покрытия ваших изменений

4. Запустите тесты локально:
   ```bash
   go test ./...
   ```

5. Зафиксируйте изменения с четкими сообщениями коммитов
   ```bash
   git commit -m "Add clear description of changes"
   ```

6. Отправьте pull request с подробным описанием изменений

### Стиль кода

- Следуйте стандартным соглашениям Go
- Используйте `gofmt` для форматирования кода
- Добавляйте комментарии для экспортируемых функций и типов
- Пишите понятные имена переменных и функций

### Тестирование

- Добавляйте тесты для новых функций
- Убедитесь, что существующие тесты проходят
- Тестируйте на разных платформах, если возможно

### Документация

- Обновляйте документацию при изменении API
- Добавляйте примеры использования для новых функций
- Поддерживайте актуальность README.md

## Процесс релиза

- Изменения отражаются в номерах версий при необходимости
- Тестируются на совместимость

## Лицензия

Внося вклад в WSSDTP, вы соглашаетесь, что ваши вклады будут лицензированы под той же лицензией, что и проект.

## Вопросы?

Не стесняйтесь открывать issue, если у вас есть вопросы или нужна разъяснение по руководству по внесению вклада.
